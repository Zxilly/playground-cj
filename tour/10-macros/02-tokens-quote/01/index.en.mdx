# Token and Tokens Types

`Tokens` is the fundamental type for Cangjie macro operations, representing a program fragment. A `Tokens` consists of multiple `Token` objects, where each `Token` is a lexical unit: an identifier, literal, keyword, or operator.

Each `Token` contains a type (`kind`, taking a `TokenKind` enum value) and content (`value`). You can manually construct any `Token` using `Token(kind, value)`, for example `Token(TokenKind.IDENTIFIER, "x")` constructs the identifier `x`.

`Tokens` supports concatenation operations: `+` can concatenate two `Tokens`, or concatenate `Tokens` with a `Token`. <CompareGroup>

<CompareWith lang="c">
C macros use text substitution; there is no concept of Token. Cannot manipulate and inspect code fragments like Cangjie.
</CompareWith>

<CompareWith lang="java">
Java's Annotation Processors can access the AST but have no direct Token representation, making code transformation more difficult.
</CompareWith>

<CompareWith lang="go">
Go does not have a macro system and cannot manipulate code fragments as data at compile time.
</CompareWith>

<CompareWith lang="rust">
Rust's procedural macros use the `TokenStream` type, which is conceptually similar but has a different API. Cangjie's `Token` explicitly includes `kind` and `value`.
</CompareWith>

</CompareGroup>

<Note>`Token` also has position information (line number, column number) used for compile error location. The `dump()` method can print complete debug information.</Note>
