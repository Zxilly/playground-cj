# 什么是宏

宏是一种编译期代码转换工具。与在运行时处理值的函数不同，宏的输入和输出都是程序代码本身。宏能读取源代码的 Token 序列，生成新的代码片段，这些生成的代码随后被编译和执行。

函数只能获取参数的运行时值，无法知道参数表达式本身。例如，函数 `f(x + y)` 只能拿到 `x + y` 的计算结果，拿不到 `x + y` 这个表达式。宏则可以访问表达式本身，实现函数做不到的功能。

仓颉提供了内置的测试宏 `@Test`、`@TestCase` 和 `@Assert`。`@Assert` 宏检查两个值是否相等，如果相等则通过测试，否则报告失败。这些宏在编译时展开为测试框架代码，简化了测试编写。

右侧代码示例展示了测试宏的基本用法。`@Test` 标记测试类，`@TestCase` 标记测试方法，`@Assert(a + b, 5)` 断言 `a + b` 的值等于 5。

<Note>宏调用使用 `@` 符号作为前缀，以区别于函数调用。</Note>

运行代码，然后将 `@Assert(a + b, 5)` 改为 `@Assert(a + b, 99)`，观察测试失败的效果。

<CompareGroup>
<CompareWith lang="c">
C 的宏通过预处理器指令 `#define` 实现，只支持文本替换，没有类型检查。仓颉的宏在编译期工作，具有类型安全。
</CompareWith>
<CompareWith lang="java">
Java 没有编译期宏系统。代码生成通过注解处理器（Annotation Processor）实现，功能有限。
</CompareWith>
<CompareWith lang="rust">
Rust 的宏也使用 `!` 后缀标记调用，如 `println!`，并能访问 AST。仓颉使用 `@` 前缀，更清晰地区分宏调用与函数调用。
</CompareWith>
<CompareWith lang="go">
Go 没有元编程宏系统。代码生成依赖外部工具如 `go:generate`。仓颉的宏在编译期内置支持，更简洁安全。
</CompareWith>
</CompareGroup>
