# 接口继承

接口可以使用 `<:` 继承一个或多个接口，形成接口层次结构。子接口继承父接口的所有成员，还可以添加新的成员。

实现子接口的类型必须同时实现子接口和所有父接口要求的成员。该类型同时成为子接口和所有父接口的子类型，可以赋值给任意父接口类型的变量。

这种"组合式"接口设计允许从简单接口构建复杂接口。一个类型可以只实现基础接口满足部分功能，也可以实现组合接口提供完整功能。

本例中，`ReadWrite` 继承了 `Readable` 和 `Writable` 接口。`FileBuffer` 实现 `ReadWrite` 后，可以赋值给 `Readable` 或 `Writable` 变量，但只能访问对应接口的方法。

<CompareGroup>

<CompareWith lang="go">
Go 不支持接口继承，但有接口嵌入（embedding）特性，可以组合其他接口。仓颉的接口继承更显式，使用 `<:` 符号让继承关系一目了然。
</CompareWith>

<CompareWith lang="java">
Java 接口使用 `extends` 继承其他接口，仓颉使用 `<:` 符号保持语法统一性。两者都支持多接口继承，允许从多个父接口组合功能。
</CompareWith>

</CompareGroup>

<Note>接口继承强调"是什么"的关系（可读可写就是 ReadWrite），而多接口实现强调"能做什么"（同时可读和可写）。选择哪种设计取决于语义需求。</Note>
