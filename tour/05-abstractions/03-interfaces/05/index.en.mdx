# Interface Inheritance

Interfaces can use `<:` to inherit from one or more interfaces, forming an interface hierarchy. A child interface inherits all members from its parent interfaces and can add new members.

Types implementing a child interface must implement members required by both the child interface and all parent interfaces. That type becomes a subtype of both the child interface and all parent interfaces, and can be assigned to variables of any parent interface type.

This "compositional" interface design allows building complex interfaces from simple ones. A type can implement only basic interfaces to satisfy partial functionality, or implement composite interfaces to provide complete functionality.

In this example, `ReadWrite` inherits from both `Readable` and `Writable` interfaces. After `FileBuffer` implements `ReadWrite`, it can be assigned to `Readable` or `Writable` variables, but can only access methods from the corresponding interface.
