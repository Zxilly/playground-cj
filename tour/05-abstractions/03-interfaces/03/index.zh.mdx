# 接口多态

接口类型变量可以持有任何实现了该接口的对象。调用方法时，实际执行的是运行时类型的具体实现，而非声明类型的方法。这就是接口多态的核心机制。

通过接口多态，可以编写通用的函数和算法，它们不依赖具体类型，只依赖接口定义的行为。这大大提高了代码的复用性和可扩展性。

接口类型的变量在编译时只能访问接口声明的方法。如果需要访问类型特有的成员，必须使用 `as` 进行类型转换。

本例中，`launch` 函数接受 `Flyable` 类型参数。传入 `Bird` 或 `Airplane` 时，虽然声明类型相同，但实际调用的方法产生不同输出，体现了运行时多态。

<CompareWith lang="go">
Go 的接口多态完全相同：接口类型变量持有具体值，方法调用在运行时动态分派。Go 称之为"鸭子类型"，仓颉的显式接口实现提供更强的编译时类型检查。
</CompareWith>

<CompareWith lang="java">
Java 的接口多态与仓颉类似，都是运行时动态绑定方法。Java 的动态分派基于虚拟方法表（vtable），仓颉采用类似的实现机制。
</CompareWith>

</CompareGroup>

<Note>多态是面向对象编程的核心特性之一。通过接口实现的代码可以轻松扩展——新增类型只需实现接口，无需修改现有代码。</Note>
