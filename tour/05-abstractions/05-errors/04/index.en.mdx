# Custom Exceptions

By inheriting from `Exception` or its subclasses, you can create custom exception types. Custom exceptions can carry additional fields, such as error codes, field names, etc., providing richer context information for error handling.

When defining custom exceptions, you need to:
- Use `open class` declaration (allows further inheritance)
- Call the parent constructor `super(message)` to pass the error message
- Override the `getClassName()` method to return the exception name
