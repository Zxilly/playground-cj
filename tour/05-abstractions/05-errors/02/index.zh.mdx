# throw 与 try-catch

`throw` 用于抛出异常，终止当前执行流程。`try-catch` 用于捕获并处理异常。抛出异常的函数不需要声明，调用方可以选择捕获或继续向上传播。

`try` 块中的代码如果抛出异常，会立即跳转到匹配的 `catch` 块。如果异常未被捕获，会继续沿调用栈向上传播，直到被处理或导致程序终止。

<CompareWith lang="go">

</CompareWith>

<CompareWith lang="java">
Java 要求方法声明可能抛出的受检异常。仓颉不区分受检和非受检异常，都不需要声明。
</CompareWith>

<CompareWith lang="rust">
Rust 使用 `Result<T, E>` 和 `?` 操作符。仓颉既支持异常也支持 `Option`，可根据场景选择。
</CompareWith>
