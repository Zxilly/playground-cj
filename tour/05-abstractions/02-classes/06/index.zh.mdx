# 抽象类

`abstract class` 定义抽象类，它可以声明没有实现的方法（抽象方法）。抽象类本身不能实例化——它是一个不完整的模板，必须由子类补全。子类必须实现所有抽象方法，否则也必须声明为抽象类。

抽象方法用 `func` 声明但不提供函数体。抽象类还可以包含具体方法（有函数体），这些具体方法可以调用抽象方法——运行时会分派到子类的实现。这种设计模式常用于定义算法框架（模板方法模式）。

下方的 `Shape` 是抽象类，声明了 `area()` 和 `label()` 两个抽象方法。具体的 `Circle` 和 `Rectangle` 必须实现它们。抽象类的 `describe()` 方法调用了这些抽象方法，形成模板方法。

<Note>
尝试用 `Shape()` 创建实例会编译报错——抽象类不可实例化。
</Note>
