# Arithmetic Operator Overloading

Operator overloading enables custom types to use operator syntax like `+`, `-`, `*`. Overloading is implemented by defining `operator func` in a `class`, `struct`, or `extend`.

The code on the right defines a `Point` class representing a 2D point, overloading the binary `+` operator (point addition) and unary `-` operator (negation). After overloading, expressions like `p1 + p2` and `-p` are natural and intuitive, consistent with built-in numeric types.

Unary operator functions have no parameters, while binary operator functions have one parameter representing the right operand. Operators don't change their inherent precedence and associativity—multiplication still takes precedence over addition.

<CompareWith lang="c">
C does not support operator overloading; while C++ does, it cannot change the behavior of built-in operators for custom types in C.
</CompareWith>

<CompareWith lang="java">
Java does not support operator overloading; libraries like BigDecimal can only use methods like `add()` and `subtract()`, lacking natural syntax.
</CompareWith>

<CompareWith lang="go">
Go does not support operator overloading; you must define regular functions like `Add(a, b Point)`.
</CompareWith>

<CompareWith lang="rust">
Rust overloads operators by implementing traits like `Add`, `Sub`, etc., with more verbose syntax but unified standards.
</CompareWith>

</CompareGroup>

<Note>Operator functions cannot be modified with `static`; they are essentially instance member functions that can access the current object using `this`.</Note>
