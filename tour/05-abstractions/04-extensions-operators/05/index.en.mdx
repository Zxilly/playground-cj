# Comparison Operator Overloading

Comparison operators (`==`, `!=`, `<`, `>`, `<=`, `>=`) are overloaded the same way as arithmetic operators, using `operator func` definitions. This enables custom types to participate in comparison operations and sorting.

The code on the right defines a `Temperature` type representing Celsius temperature, overloading all six comparison operators. After overloading, expressions like `boiling > freezing` and `body == another` can be used directly, making the code more intuitive and readable.

Comparison operators should typically be implemented in pairs: if you implement `<`, you should implement `>`; if you implement `==`, you should implement `!=`. Maintaining semantic consistency is important to avoid confusing behavior.

<CompareGroup>

<CompareWith lang="c">
In C, `==` and other comparison operators perform byte-level comparison for custom types and cannot be overloaded; manual comparison functions like `compare()` are usually written instead.
</CompareWith>

<CompareWith lang="java">
Java implements comparison by overriding `equals()` and `compareTo()` methods, but `==` still performs reference comparison only, which can be confusing.
</CompareWith>

<CompareWith lang="go">
Go requires implementing the `Comparable` interface or defining methods like `Less`; it does not support directly overloading operators like `==`.
</CompareWith>

<CompareWith lang="rust">
Rust implements comparison functionality through traits like `PartialEq`, `Ord`, etc., with stricter specification requirements.
</CompareWith>

</CompareGroup>

<Note>The return type of comparison operators should be `Bool`, which is a semantic convention.</Note>

Try implementing only `==` and `<`, and see if other comparison operations are automatically derived (hint: they won't be).
