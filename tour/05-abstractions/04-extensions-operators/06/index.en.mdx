# Subscript Operator Overloading

Overloading the `[]` operator allows custom types to support array-like index access syntax. The subscript operator has two forms: the getter form `arr[i]` and the setter form `arr[i] = value`, distinguished by whether a named parameter `value!` exists.

The code on the right defines a `Grid` class representing a 2D table. The getter form `operator func [](row, col)` returns the cell value; the setter form `operator func [](row, col, value!)` takes a special named parameter `value!` (note the exclamation mark), and the return type must be `Unit`.

The subscript operator supports multiple index parameters, not limited to single-dimensional arrays. When calling, you don't need to write the named parameter form `grid[row, col, value: v]`, just use `grid[row, col] = v` directly.

<CompareGroup>

<CompareWith lang="c">
In C, `[]` only works on array pointers and cannot be overloaded for custom types; functions like `grid_get()` and `grid_set()` are used instead.
</CompareWith>

<CompareWith lang="java">
Java does not support operator overloading and must use `get()` and `set()` methods; the Collections Framework provides these methods but cannot change the syntax.
</CompareWith>

<CompareWith lang="go">
Go does not support subscript operator overloading, but you can define methods like `At(i, j)` for types to simulate indexed access.
</CompareWith>

<CompareWith lang="rust">
Rust implements subscript access through the `Index` and `IndexMut` traits, separating immutable and mutable access.
</CompareWith>

</CompareGroup>

<Note>The `value` in the setter form is a special marker; when calling, you don't need to pass a named parameter, just use the `arr[i] = value` syntax directly.</Note>

Try adding boundary checking to print an error message instead of crashing when the index is out of bounds.

CJ:
class Grid {
    private let rows: Int64
    private let cols: Int64
    private var data: Array<Int64>

    public init(rows: Int64, cols: Int64) {
        this.rows = rows
        this.cols = cols
        this.data = Array<Int64>(rows * cols, {_ => 0})
    }

    // Getter form: grid[row, col]
    public operator func [](row: Int64, col: Int64): Int64 {
        return this.data[row * this.cols + col]
    }

    // Setter form: grid[row, col] = value
    public operator func [](row: Int64, col: Int64, value!: Int64): Unit {
        this.data[row * this.cols + col] = value
    }

    // Display grid contents
    public func display(): Unit {
        println("Grid contents (${this.rows} x ${this.cols}):")
        var r: Int64 = 0
        while (r < this.rows) {
            var line = ""
            var c: Int64 = 0
            while (c < this.cols) {
                if (c > 0) {
                    line = line + "\t"
                }
                line = line + "${this[r, c]}"
                c += 1
            }
            println(line)
            r += 1
        }
    }

    // Get number of rows
    public prop numRows: Int64 {
        get() { this.rows }
    }

    // Get number of columns
    public prop numCols: Int64 {
        get() { this.cols }
    }
}

main() {
    let grid = Grid(3, 3)

    // Use subscript operator to assign values
    grid[0, 0] = 1
    grid[0, 1] = 2
    grid[0, 2] = 3
    grid[1, 0] = 4
    grid[1, 1] = 5
    grid[1, 2] = 6
    grid[2, 0] = 7
    grid[2, 1] = 8
    grid[2, 2] = 9

    // Display entire grid
    grid.display()
    println()

    // Use subscript operator to get values
    println("Center element grid[1,1] = ${grid[1, 1]}")
    println("Top-left corner grid[0,0] = ${grid[0, 0]}")
    println("Bottom-right corner grid[2,2] = ${grid[2, 2]}")
    println()

    // Modify an element
    println("Changing grid[1,1] to 99")
    grid[1, 1] = 99
    println("After modification grid[1,1] = ${grid[1, 1]}")
}
