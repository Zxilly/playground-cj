# 限制注解的使用位置

默认情况下，自定义注解可以用在类型声明、函数、成员变量、参数等各种位置。但有时我们希望限制注解只能用在特定位置，以减少误用。这可以通过在定义注解时指定 `target` 参数来实现。

`target` 参数接受一个 `Array<AnnotationKind>` 数组，其中 `AnnotationKind` 是标准库中的枚举，包含 `Type`、`MemberFunction`、`MemberProperty`、`Parameter` 等值。当限定 target 后，注解只能用在声明的位置列表中。

右侧代码定义了只能用于成员函数的 `@Log` 注解，以及只能用于类型的 `@Entity` 注解，并展示了它们的正确用法。

<CompareWith lang="java">

<CompareWith lang="rust">

</CompareWith>

</CompareGroup>

<Note>当没有指定 target 时，注解可以用在所有支持的位置，包括类型、函数、变量、参数等。</Note>
