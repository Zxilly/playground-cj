# cjpm init — 创建项目

`cjpm init` 命令用于初始化一个新的仓颉项目。它会创建标准的项目结构和配置文件。

执行 `cjpm init` 后，会生成以下目录结构：一个 `cjpm.toml` 配置文件，一个 `src/` 源码目录，以及一个默认的 `src/main.cj` 入口文件。`cjpm.toml` 是项目的核心配置文件，用于定义项目名称、版本、依赖等信息。

配置文件中必须包含 `[package]` 部分，其中 `name` 是模块名，`version` 是模块版本号，`output-type` 指定编译产物类型（如 `executable` 表示可执行文件）。

<Note>`[package]` 和 `[workspace]` 字段不能在同一个 `cjpm.toml` 文件中共存。单模块用 `[package]`，多模块用 `[workspace]`。</Note>
