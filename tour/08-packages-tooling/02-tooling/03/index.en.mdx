# cjpm build, run, and test

`cjpm build` compiles the current project. It first checks dependencies, then calls `cjc` to build. Artifacts are placed in the `target/release/` directory by default. Use the `-V` option to view detailed compilation commands.

`cjpm run` is a more convenient command that first compiles the project and then runs it directly, suitable for rapid iteration during development.

`cjpm test` is used to run unit tests. When used with `@Test` and `@TestCase` annotations, it can automatically discover and execute test cases.

<Note>`cjpm build` supports incremental compilation, only recompiling modified files to speed up builds.</Note>
