# cjpm build、run 与 test

`cjpm build` 编译当前项目。它会先检查依赖项，然后调用 `cjc` 进行构建，产物默认放在 `target/release/` 目录下。使用 `-V` 选项可以查看详细的编译命令。

`cjpm run` 是更便捷的命令，它会先编译项目，然后直接运行，适合开发时快速迭代。

`cjpm test` 用于运行单元测试。配合 `@Test` 和 `@TestCase` 注解使用，可以自动发现并执行测试用例。

<Note>`cjpm build` 支持增量编译，只重新编译修改过的文件，加快构建速度。</Note>
