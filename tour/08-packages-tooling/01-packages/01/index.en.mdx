# Package Declaration

Every Cangjie source file begins with a `package` declaration that specifies which package the file belongs to. The package declaration must be the first non-comment line in the file.

The package name needs to reflect the path of the source file relative to the project's `src` directory. For example, a file located under `src/myapp/utils/` should have the package declaration `package myapp.utils`. Package names are separated by dots, and folder names must match the package name components.

Files located in the `src` root directory can omit the package declaration, and the compiler will automatically assign them the default package name `default`. This is convenient for small projects or rapid prototyping.
