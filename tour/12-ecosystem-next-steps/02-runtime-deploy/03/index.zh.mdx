# 编译后端：CJNative 与 CJVM

仓颉支持两种编译后端：**CJNative** 和 **CJVM**。CJNative 后端将代码编译为原生机器码，直接在操作系统上运行，具有最佳性能和最小内存占用。CJVM 后端将代码编译为 JVM 字节码，可以与 Java 生态无缝互操作，复用 Java 库和框架。

两种后端的源代码语法相同，只需选择对应的 SDK 即可切换。使用 CJNative SDK 编译得到原生二进制，使用 CJVM SDK 则编译得到 JVM 字节码（`.class` 文件）。同一份代码可以在不同后端间移植。

编写跨后端代码时，可以使用条件编译注解 `@When[backend == "cjnative"]` 来区分不同后端的实现。后端条件变量 `backend` 支持 `==` 和 `!=` 操作，目前可选值为 `cjnative` 和 `cjvm`。

<CompareGroup>

<CompareWith lang="c">
C 只编译为原生代码。编译后的代码与目标平台高度耦合，无法跨平台运行。
</CompareWith>

<CompareWith lang="java">
Java 编译为平台无关的字节码，需要 JVM 解释或编译执行。仓颉的 CJVM 后端试图融合这一特性。
</CompareWith>

<CompareWith lang="rust">
Rust 只编译为原生代码，没有 JVM 后端。仓颉提供两个后端，兼顾原生性能和 Java 生态兼容。
</CompareWith>

<CompareWith lang="go">
Go 只支持原生编译。Kotlin 才同时支持 JVM 和原生（Native），类似仓颉的双后端策略。
</CompareWith>

</CompareGroup>

<Note>CJVM 后端支持正在持续完善中，具体功能范围请参考官方文档。本教程默认使用 CJNative 后端。</Note>
