# Deploying Cangjie Runtime

Cangjie executable programs depend on runtime libraries to run properly. The runtime provides core functionality such as memory management, garbage collection, and exception handling. By default, Cangjie uses dynamic linking, and the generated executable files need to load Cangjie runtime dynamic libraries at runtime.

When deploying, you need to ensure that runtime libraries exist on the target machine. On Linux, set the `LD_LIBRARY_PATH` environment variable to point to the `runtime/lib/linux_x86_64_cjnative/` directory in the SDK; on macOS, use `DYLD_LIBRARY_PATH`; on Windows, add the runtime directory to `PATH`.

If you want to generate a self-contained executable file, you can use the `--static` option (Linux platform only). Static linking compiles the runtime into the binary file, eliminating the need to deploy the runtime separately in the target environment, but it will increase the file size.

<CompareGroup>

<CompareWith lang="c">
C dynamically links the standard library by default. Static linking requires explicitly specifying the `-static` option. Cangjie's linking strategy is similar.
</CompareWith>

<CompareWith lang="java">
Java applications are packaged as JAR files and require the JVM and Java runtime. There is no concept of static linking.
</CompareWith>

<CompareWith lang="go">
Go statically links by default, generating completely independent binary files. Cangjie dynamically links by default and requires runtime deployment or using `--static`.
</CompareWith>

<CompareWith lang="rust">
Rust standard library statically links by default, generating independent binary files. Cangjie can choose between dynamic or static linking.
</CompareWith>

</CompareGroup>

<Note>Static linking only applies to the Linux platform. macOS and Windows platforms currently only support dynamic linking.</Note>

Try running the program on the right to check if your runtime environment is configured correctly. If the runtime is not configured, the program will report an error indicating that the dynamic library cannot be found.

CJ:
// Dynamic linking compilation (default):
//   cjc runtime_check.cj -o runtime_check
//   export LD_LIBRARY_PATH=$CANGJIE_HOME/runtime/lib/linux_x86_64_cjnative:$LD_LIBRARY_PATH
//   ./runtime_check
//
// Static linking compilation (Linux only):
//   cjc runtime_check.cj -o runtime_check --static
//   ./runtime_check

main() {
    println("=== Cangjie Runtime Check ===")
    
    // Test basic data type operations
    let nums = [10, 20, 30, 40, 50]
    var total = 0
    for (n in nums) {
        total += n
    }
    println("Sum of array elements: ${total}")
    
    // Test string operations (depends on runtime)
    let text = "Cangjie Runtime"
    println("String length: ${text.size}")
    println("String content: ${text}")
    
    // Test object creation (depends on memory management)
    class Point {
        let x: Int64
        let y: Int64
        init(x: Int64, y: Int64) {
            this.x = x
            this.y = y
        }
    }
    let p = Point(100, 200)
    println("Coordinate point: (${p.x}, ${p.y})")
    
    println("Runtime check complete, everything is normal!")
}
