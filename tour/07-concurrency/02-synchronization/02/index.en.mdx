# Atomic Operations

For simple concurrent access to integers, booleans, or reference types, atomic operations provide a lock-free, efficient solution. Atomic operations are directly supported by hardware and don't require explicit lock mechanisms.

Cangjie provides various atomic types, such as `AtomicInt64`, `AtomicBool`, and `AtomicReference`. Atomic operations for integer types include `load()` for reading, `store()` for writing, `fetchAdd()` for atomic addition, `compareAndSwap()` for compare-and-swap, and more.

Atomic operations are suitable for simple scenarios like counting and flagging. If you need to protect compound operations on multiple variables, or if the critical section contains complex logic, you should use `Mutex` instead of atomic operations.

<CompareGroup>

<CompareWith lang="c">
C relies on compiler built-in functions like `__sync_` or `__atomic_` for atomic operations, which have inconsistent APIs and are difficult to use.
</CompareWith>

<CompareWith lang="java">
Java provides the `java.util.concurrent.atomic` package with types like `AtomicInteger`, with APIs similar to Cangjie, but memory ordering options are complex.
</CompareWith>

<CompareWith lang="go">
Go's `sync/atomic` package provides functional APIs, such as `atomic.AddInt64(&addr, delta)`. Cangjie's atomic types are object-oriented, with methods bound to atomic objects.
</CompareWith>

<CompareWith lang="rust">
Rust provides atomic types through the `std::sync::atomic` module, using an `Ordering` parameter to control memory ordering. Cangjie currently only supports sequential consistency, making it simpler to use.
</CompareWith>
