# Combining Synchronization Mechanisms

In real concurrent programs, it's often necessary to combine multiple synchronization mechanisms. Understanding when to use `Mutex` and when to use atomic operations is key to writing efficient concurrent code.

A practical rule of thumb: for single integers or booleans, prefer atomic operations (better performance); for complex data structures or scenarios requiring protection of multiple variables, use `Mutex` (more flexible and safer).

The code on the right demonstrates a complete concurrent computing scenario: multiple workers calculate squares in parallel, write formatted results to a shared list (protected by `Mutex`), while using an atomic counter to track completion count. This is a simplified version of the classic producer-consumer pattern.

<Note>When choosing synchronization mechanisms, consider correctness first, then performance. Premature optimization may lead to hard-to-detect data race bugs.</Note>

CJ:
import std.sync.*
import std.collection.ArrayList

// Function to calculate square value
func square(n: Int64): Int64 {
    return n * n
}

main(): Int64 {
    // Define task list
    let jobs = [2, 4, 6, 8, 10]
    
    // Shared data structures
    let results = ArrayList<String>()
    let resultsLock = Mutex()
    let completed = AtomicInt64(0)
    
    // Store all Futures
    let futures = ArrayList<Future<Unit>>()
    
    // Start a thread for each task
    for (job in jobs) {
        let current = job
        let future = spawn {
            // Execute calculation (no synchronization needed)
            let value = square(current)
            
            // Write result to shared list (Mutex required)
            synchronized(resultsLock) {
                results.add("${current}² = ${value}")
            }
            
            // Update completion count (use atomic operation)
            let _ = completed.fetchAdd(1)
        }
        futures.add(future)
    }
    
    // Wait for all threads to complete
    for (future in futures) {
        future.get()
    }
    
    // Output results
    println("Completion count: ${completed.load()}")
    println("Calculation results:")
    for (line in results) {
        println("  ${line}")
    }
    
    return 0
}
