# Combining Synchronization Mechanisms

In real concurrent programs, it's often necessary to combine multiple synchronization mechanisms. Understanding when to use `Mutex` and when to use atomic operations is key to writing efficient concurrent code.

A practical rule of thumb: for single integers or booleans, prefer atomic operations (better performance); for complex data structures or scenarios requiring protection of multiple variables, use `Mutex` (more flexible and safer).

The code on the right demonstrates a complete concurrent computing scenario: multiple workers calculate squares in parallel, write formatted results to a shared list (protected by `Mutex`), while using an atomic counter to track completion count. This is a simplified version of the classic producer-consumer pattern.
