# 互斥锁 Mutex

当多个线程同时访问和修改共享数据时，会产生数据竞争（data race），导致不可预测的结果。互斥锁通过确保同一时刻只有一个线程能进入临界区来解决这个问题。

仓颉使用 `Mutex` 类型作为可重入互斥锁。可重入意味着同一个线程可以多次获取同一把锁而不会死锁。对 `Mutex` 的基本操作包括 `lock()` 获取锁和 `unlock()` 释放锁。

为了简化锁的管理，仓颉提供了 `synchronized` 关键字。`synchronized(mtx)` 会在进入代码块时自动获取锁，退出时自动释放锁，即使发生异常或提前返回也能正确释放。

<CompareGroup>

<CompareWith lang="c">
C 中需要使用 `pthread_mutex_t` 并手动调用 `pthread_mutex_lock()` 和 `pthread_mutex_unlock()`，容易遗漏 unlock 导致死锁。
</CompareWith>

<CompareWith lang="java">
Java 可以用 `synchronized` 关键字或显式的 `ReentrantLock`，但 `ReentrantLock` 需要手动 `lock()` 和 `unlock()`；仓颉的 `synchronized` 自动管理。
</CompareWith>

<CompareWith lang="go">
Go 使用 `sync.Mutex` 的 `Lock()` 和 `Unlock()` 方法，通常配合 `defer` 确保释放。仓颉的 `synchronized` 类似于 Java 的用法，自动管理锁的生命周期。
</CompareWith>

<CompareWith lang="rust">
Rust 的 `Mutex<T>` 将数据和锁封装在一起，通过 `lock()` 返回 `MutexGuard`，利用 RAII 自动释放。仓颉的 `Mutex` 是独立的锁对象，不封装数据。
</CompareWith>
