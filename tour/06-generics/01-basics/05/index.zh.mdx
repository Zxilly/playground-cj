# 泛型枚举

泛型枚举的构造器可以携带类型化数据。类型参数在枚举名称后的尖括号中声明，然后可以在构造器的参数类型中使用。这允许枚举在不同情况下携带不同类型的数据。

`Result<T, E>` 用 `Ok(T)` 表示成功并携带值，用 `Err(E)` 表示失败并携带错误信息。两个类型参数分别对应成功值和错误值的类型，调用方可以自由选择。`Option<T>` 也是泛型枚举的经典例子：`Some(T)` 或 `None`。

对泛型枚举进行模式匹配时，可以提取携带的值并保留完整的类型信息。匹配分支中的变量会被推断为相应的类型，无需手动转换。这种设计使得错误处理既类型安全又表达力强。

<CompareGroup>

<CompareWith lang="c">
C 中的联合（union）无法携带类型信息，通常需要额外的标记字段来区分数据类型，而且不支持泛型。
</CompareWith>

<CompareWith lang="java">
Java 没有真正的泛型枚举；虽然可以使用泛型类包装枚举，但无法在枚举层级实现泛型约束。
</CompareWith>

<CompareWith lang="go">
Go 没有内置的泛型枚举，通常使用 `interface{}` 或自定义类型来模拟。
</CompareWith>

<CompareWith lang="rust">
Rust 的泛型枚举语法完全相同：`enum Result<T, E> { Ok(T), Err(E) }`，这是标准库中的核心类型。
</CompareWith>

</CompareGroup>

<Note>泛型枚举的每个构造器可以选择性地携带数据，有的构造器可以携带类型化数据，有的可以不携带。</Note>
