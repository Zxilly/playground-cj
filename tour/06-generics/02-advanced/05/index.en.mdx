# Generic Registry

`Registry<T>` synthesizes multiple concepts learned in this chapter: type parameter `T`, `where T <: Identifiable` constraint, `HashMap` storage, and `Option<T>` return value. The constraint ensures each element has a `getId()` method for unique identification.

This pattern—a generic container with constraints—is common in real-world Cangjie code. Through constraints, generic classes can safely manipulate elements without knowing their concrete types; through generics, instantiated containers maintain specific element types, avoiding runtime type checks.

The code on the right implements a user registry: it requires stored elements to implement the `Identifiable` interface, uses `HashMap` internally for storage, and returns `Option<T>` for queries to elegantly handle "not found" cases.
