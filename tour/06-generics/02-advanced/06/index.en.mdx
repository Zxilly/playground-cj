# const Functions and Compile-Time Evaluation

`const func` is a special type of function that can be evaluated at compile time. When all its parameters are compile-time constants, the function executes during the compilation phase, and the result is directly embedded in the program. This reduces runtime computation overhead.

Expressions in `const` function bodies must be `const` expressions and cannot include I/O operations, modifying global state, or other side effects. You can declare local variables with `let` (not `var`), and use control flow like `if`, `match`, and `try`.

Structs and classes can define `const init` constructors to support compile-time construction. <CompareGroup>

<CompareWith lang="c">
C has no compile-time function evaluation; you must use preprocessor macros `#define` or runtime calculations, sacrificing flexibility and type safety.
</CompareWith>

<CompareWith lang="java">
Java does not support compile-time function evaluation; while it has compile-time constants (`static final`), arbitrary computations cannot be executed and must be hard-coded.
</CompareWith>

<CompareWith lang="go">
Go does not have compile-time evaluation mechanisms; all functions execute at runtime, though the compiler does constant folding optimization.
</CompareWith>

<CompareWith lang="rust">
Rust's `const fn` has similar functionality, also supporting compile-time evaluation with similar syntax and restrictions.
</CompareWith>

</CompareGroup>

<Note>`const` functions don't always execute at compile time. When called in non-`const` contexts, they execute at runtime like normal functions.</Note>
