# 多重约束

`where` 子句可以用 `&` 连接多个约束，要求类型参数同时满足所有接口。`where T <: Comparable<T> & Displayable` 表示 `T` 必须同时实现 `Comparable` 和 `Displayable` 两个接口。

多重约束让你可以在同一个函数中对值进行多种操作。比如，约束 `T` 既是可比较的又是可显示的，就可以同时进行大小比较和格式化输出，而不损失类型安全。

右侧代码定义了 `Score` 类同时实现两个接口，然后使用双重约束的 `showMax` 函数：它可以找出较大的值并格式化输出。注意 `showMax` 内部调用了只要求 `Comparable` 的 `findMax` 函数，约束条件可以按需叠加。

<CompareGroup>

<CompareWith lang="c">
C 无法表达多重约束，必须手动检查每个操作的可行性，容易出错且无编译时保证。
</CompareWith>

<CompareWith lang="java">
Java 的多重约束使用 `<T extends Comparable & Displayable>`，但继承链中只能有一个类，其余必须是接口。
</CompareWith>

<CompareWith lang="go">
Go 使用 `interface { Comparable; Displayable }` 组合多个接口作为约束。
</CompareWith>

<CompareWith lang="rust">
Rust 使用 `where T: Comparable + Displayable` 语法，同样用 `+` 连接多个 trait。
</CompareWith>

</CompareGroup>

<Note>多个约束之间是"且"的关系，类型参数必须实现所有约束的接口。</Note>
