# Assignment Operators

The assignment operator `=` is used to assign a new value to a variable declared with `var`. Unlike many languages, the type of an assignment expression in Cangjie is `Unit`, and its value is `()`. This means assignment expressions cannot be used as conditions, and the compiler helps you avoid the common mistake of writing `=` instead of `==`.

Cangjie supports compound assignment operators that combine arithmetic operations with assignment: `+=`, `-=`, `*=`, `/=`, `%=`, `**=`, etc. For example, `score += 5` is equivalent to `score = score + 5`, but more concise. Compound assignment operators require both operands to be of the same type.

Cangjie also supports multi-assignment expressions, allowing you to assign tuple values to multiple variables at once. For example, `(a, b) = (b, a)` can conveniently swap the values of two variables.

C's assignment expressions have a return value (the value after assignment), which can be used in conditional statements. This can easily lead to the common mistake of writing `=` instead of `==`. Cangjie's design is safer.

<CompareWith lang="java">
Java's assignment also has a return value and can be used in conditions, similarly prone to confusing `=` with `==`. Java supports compound assignment operators like `+=`.
</CompareWith>

<CompareWith lang="go">
Go's assignment expressions also have a return value, but using assignment in conditional statements results in a compilation error. Go also supports compound assignment operators like `+=`. Go doesn't have multi-assignment syntax, but supports `a, b = b, a` for multiple return value assignment.
</CompareWith>

<CompareWith lang="rust">
Rust's assignment expressions return `()`, just like Cangjie. Rust also supports compound assignment operators, but doesn't support tuple destructuring assignment like `(a, b) = (b, a)`; you need to use `std::mem::swap` or a temporary variable.
</CompareWith>
