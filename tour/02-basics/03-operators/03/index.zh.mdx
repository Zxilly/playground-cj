# 赋值运算符

赋值运算符 `=` 用于给 `var` 声明的变量赋予新值。与许多语言不同，仓颉的赋值表达式类型是 `Unit`，值是 `()`。这意味着赋值表达式不能作为条件使用，编译器会帮你避免将 `==` 误写成 `=` 的常见错误。

仓颉支持复合赋值运算符，将算术运算与赋值组合在一起：`+=`、`-=`、`*=`、`/=`、`%=`、`**=` 等。例如，`score += 5` 等价于 `score = score + 5`，但更简洁。复合赋值运算符要求两个操作数类型相同。

仓颉还支持多赋值表达式，可以一次性将元组的值赋给多个变量，例如 `(a, b) = (b, a)` 可以方便地交换两个变量的值。

<CompareGroup>

<CompareWith lang="c">
C 的赋值表达式有返回值（赋值后的值），可以在条件语句中使用，这容易导致误将 `=` 写成 `==` 的错误。仓颉的设计更安全。
</CompareWith>

<CompareWith lang="java">
Java 的赋值也有返回值，可以在条件中使用，同样容易导致 `=` 和 `==` 的混淆。Java 支持复合赋值运算符如 `+=`。
</CompareWith>

<CompareWith lang="go">
Go 的赋值表达式也有返回值，但在条件语句中使用赋值会编译错误。Go 同样支持 `+=` 等复合赋值运算符。Go 没有多赋值语法，但支持 `a, b = b, a` 的多返回值赋值。
</CompareWith>

<CompareWith lang="rust">
Rust 的赋值表达式返回 `()`，与仓颉相同。Rust 也支持复合赋值运算符，但不支持 `(a, b) = (b, a)` 这种元组解构赋值，需要使用 `std::mem::swap` 或临时变量。
</CompareWith>
