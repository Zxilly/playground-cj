# 注释

仓颉支持两种注释形式。`//` 开始单行注释，从 `//` 到行末的内容会被编译器忽略。`/* */` 包围多行注释，可以跨越多行。

仓颉的多行注释支持嵌套：可以在一个多行注释内包含另一个多行注释。这与其他一些语言（如 Go）不同，在需要临时注释掉一段包含多行注释的代码时非常方便。右侧示例展示了嵌套注释的用法。

注释不会影响程序的执行，但能显著提升代码的可读性。良好的注释习惯是编写可维护代码的关键。

<Note>注释应该解释"为什么"而不是"做什么"。代码本身已经说明了在做什么，注释应该解释设计决策和原因。</Note>

<CompareGroup>

<CompareWith lang="c">
C 的多行注释 `/* */` 不支持嵌套，注释掉包含注释的代码时容易出错；仓颉支持嵌套注释。
</CompareWith>

<CompareWith lang="java">
Java 同样支持 `//` 和 `/* */` 注释，但 `/* */` 也不支持嵌套，与仓颉不同。
</CompareWith>

<CompareWith lang="go">
Go 的多行注释不能嵌套，这是仓颉与 Go 的重要区别。仓颉的嵌套注释在调试时更方便。
</CompareWith>

<CompareWith lang="rust">
Rust 的注释语法与仓颉完全相同：`//` 用于单行注释，`/* */` 用于多行注释，且都支持嵌套。
</CompareWith>
