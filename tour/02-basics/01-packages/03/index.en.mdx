# Comments

Cangjie supports two forms of comments. `//` starts a single-line comment, where content from `//` to the end of the line is ignored by the compiler. `/* */` surrounds multi-line comments, which can span multiple lines.

Cangjie's multi-line comments support nesting: you can include another multi-line comment within a multi-line comment. This is different from some other languages (like Go), and is very convenient when you need to temporarily comment out a section of code that contains multi-line comments. The example on the right demonstrates the use of nested comments.

Comments don't affect program execution but can significantly improve code readability. Good commenting habits are key to writing maintainable code.

<Note>Comments should explain "why" rather than "what". The code itself already shows what it's doing; comments should explain design decisions and reasoning.</Note>
