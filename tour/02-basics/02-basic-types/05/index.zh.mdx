# 字符串与插值

字符串类型用 `String` 表示，是由 Unicode 字符组成的序列。字符串字面量分为单行字符串、多行字符串和多行原始字符串三种形式。

单行字符串用一对单引号或双引号包围。多行字符串用三个单引号 `'''` 或三个双引号 `"""` 包围，可以跨越多行。多行原始字符串以 `#` 和引号开头，内容保持原样，转义字符不会被转义。

插值字符串通过 `${expression}` 将表达式的值嵌入字符串中。花括号内可以是任意表达式，甚至可以包含声明语句。插值字符串让字符串拼接变得简洁直观，避免了繁琐的 `+` 操作。右侧代码展示了字符串的基本使用和插值功能。

<Note>多行原始字符串不支持下划线插值，其中的 `${...}` 会被当作原始内容。</Note>

<CompareGroup>

<CompareWith lang="c">
C 没有原生字符串类型，需要 `char[]` 数组和字符串库函数；仓颉的 `String` 是第一类型，内建插值支持。
</CompareWith>

<CompareWith lang="java">
Java 的 `String` 不可变，拼接用 `+` 或 `StringBuilder`；仓颉的插值语法 `${expr}` 更简洁，避免临时对象。
</CompareWith>

<CompareWith lang="go">
Go 使用反引号 `` ` `` 表示原始字符串（无转义），双引号表示普通字符串。Go 的字符串插值需要用 `fmt.Sprintf`，仓颉直接用 `${expr}` 更简洁。Go 用 `+` 拼接字符串，仓颉既可以用 `+` 也可以用插值。
</CompareWith>

<CompareWith lang="rust">
Rust 的字符串插值需要用 `format!` 宏，如 `format!("值是 {}", x)`。仓颉的内建插值语法 `${expr}` 更接近 JavaScript 或 Swift 的风格，更直观简洁。
</CompareWith>
