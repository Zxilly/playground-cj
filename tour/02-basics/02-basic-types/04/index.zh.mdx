# 字符类型

字符类型用 `Rune` 表示，可以表示 Unicode 字符集中的任意字符。一个 `Rune` 字面量以字符 `r` 开头，后跟一对单引号或双引号包围的字符。

字符字面量有三种形式：单个字符（如 `r'A'`）、转义字符（如 `r'\n'`、`r'\t'`）、通用字符（如 `r'\u{4f60}'`，后面跟 1-8 位十六进制数表示 Unicode 码点）。转义字符包括换行 `\n`、制表符 `\t`、反斜杠 `\\` 等。

字符类型支持关系操作符进行比较，比较的是字符的 Unicode 码点值。`Rune` 可以转换为 `UInt32`，整数类型也可以转换为 `Rune`（必须落在有效的 Unicode 范围内）。右侧代码展示了字符的基本使用和转义字符。

<Note>`Rune` 表示单个 Unicode 字符，与字符串 `String` 不同。一个 Unicode 字符可能由多个 UTF-16 编码单元组成，但 `Rune` 始终表示一个完整的码点。</Note>

<CompareGroup>

<CompareWith lang="c">
C 的字符 `char` 只能表示单字节字符，无法处理 Unicode；仓颉的 `Rune` 原生支持任意 Unicode 字符。
</CompareWith>

<CompareWith lang="java">
Java 的 `char` 基于 UTF-16，可表示基本多文种平面的字符；仓颉的 `Rune` 更简单，直接表示 Unicode 码点。
</CompareWith>

<CompareWith lang="go">
Go 的 `rune` 是 `int32` 的别名，表示一个 Unicode 码点。仓颉的 `Rune` 是独立的字符类型，需要用 `r` 前缀。Go 用单引号 `'x'` 表示 rune，仓颉用 `r'x'`。
</CompareWith>

<CompareWith lang="rust">
Rust 的 `char` 类型也是表示 Unicode 码点，占用 4 字节。仓颉的 `Rune` 与之类似，但字面量语法不同（Rust 用单引号 `'字'`，仓颉用 `r'字'`）。
</CompareWith>

</CompareGroup>
