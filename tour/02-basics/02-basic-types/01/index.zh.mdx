# 整数类型

整数类型分为有符号（如 `Int8`、`Int64`）和无符号（如 `UInt8`、`UInt64`）两大类，数字后缀表示位数。`Int64` 是默认的整数类型——当整数字面量没有类型上下文时，编译器会推断为 `Int64`。

整数字面量支持四种进制表示：十进制（无前缀）、二进制（`0b` 或 `0B`）、八进制（`0o` 或 `0O`）、十六进制（`0x` 或 `0X`）。可以用下划线 `_` 作为分隔符提高可读性，如 `1_000_000`。

可以通过后缀明确指定字面量的类型：`i8` 表示 `Int8`，`u64` 表示 `UInt64`，以此类推。右侧代码展示了不同进制和类型的整数字面量。

<Note>使用 `Int64` 是首选，因为它的范围足够大（-2^63 到 2^63-1），避免了不必要的类型转换。</Note>

<CompareWith lang="c">

<CompareWith lang="java">
Java 的整数类型大小固定（`int` 32 位、`long` 64 位）；仓颉提供更多选择，包括 `Int8`、`Int16` 等小类型。
</CompareWith>

<CompareWith lang="go">
Go 的 `int` 和 `uint` 大小取决于平台，仓颉的 `IntNative` 和 `UIntNative` 与之类似。Go 用 `0x` 表示十六进制但没有八进制字面量前缀（Go 1.13 之前），仓颉明确使用 `0o` 前缀。
</CompareWith>

<CompareWith lang="rust">
Rust 的整数默认推断为 `i32`，仓颉默认推断为 `Int64`。两者都支持类型后缀，但 Rust 用 `0o` 而仓颉用 `0o` 或 `0O`。
</CompareWith>
