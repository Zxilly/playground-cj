# Implicit Returns

In Cangjie, the last expression in a function body is automatically returned without explicitly writing the `return` keyword. This style makes the code more concise, especially for simple calculation functions.

The type of the function body depends on the type of the last expression: if the last item is an expression, the function body type is that expression's type; if the last item is a variable definition or function declaration, the function body type is `Unit`. The compiler infers the return value based on the function body type.

The code on the right shows three functions using implicit returns: `multiply` returns the product of two numbers, `isPositive` checks if a number is positive, and `describe` returns a description string. The last expression of each function is the return value.

<CompareGroup>

<CompareWith lang="c">
C requires explicit `return` statements and doesn't support implicit returns. Function bodies typically end with `return value;`.
</CompareWith>

<CompareWith lang="java">
Java also requires explicit `return` statements and cannot omit them. Java methods must explicitly specify the return type and don't support implicit returns.
</CompareWith>

<CompareWith lang="go">
Go doesn't support implicit returns and requires explicit `return` statements. Cangjie's implicit returns are similar to Rust and Ruby, reducing boilerplate code.
</CompareWith>

<CompareWith lang="rust">
Rust also supports implicit returns, where the value of the last expression in a function is the return value. The syntax is nearly identical in both, and neither requires a trailing semicolon (adding one turns it into a statement, returning `()`).
</CompareWith>

</CompareGroup>

<Note>Don't add a semicolon after an implicitly returned expression, or it becomes a statement and the return type becomes `Unit`.</Note>
