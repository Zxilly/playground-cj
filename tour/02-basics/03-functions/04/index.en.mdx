# Named Parameters

In Cangjie, you can define named parameters using the `name!: Type` syntax. When calling a function, named parameters require writing the parameter name in the format `name: value`. Named parameters can be passed in any order, which makes code more readable and reduces parameter passing errors.

Named parameters must be marked with `!`, which is Cangjie's unique syntax. Parameter lists can contain both regular parameters and named parameters, but named parameters must come after regular parameters. When calling, regular parameters are passed by position, while named parameters are passed by name.

<CompareWith lang="java">
Java also doesn't support named parameters and requires positional argument passing. Some Java frameworks (like Lombok) provide the Builder pattern to improve this, but it's more complex than Cangjie's built-in support.
</CompareWith>

<CompareWith lang="go">
Go doesn't support named parameter syntax and can only pass arguments by position. Cangjie's named parameters make function calls clearer, similar to Python's `func(name=value)` syntax.
</CompareWith>

<CompareWith lang="rust">
Rust doesn't support named parameters but can simulate them through structs or the `#[derive(Builder)]` pattern. Cangjie's named parameters are a built-in language feature, more concise and intuitive.
</CompareWith>

</CompareGroup>

<Note>Named parameters must be defined after regular parameters. When calling, named parameters can be in any order.</Note>
