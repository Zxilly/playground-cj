# 多返回值

仓颉的函数可以通过返回元组来实现"多返回值"的效果。元组使用 `(T1, T2, ...)` 表示，可以把多个不同类型的值组合在一起。调用方可以通过索引 `tuple[0]`、`tuple[1]` 访问元素，也可以用解构语法 `let (a, b) = tuple` 直接提取到独立变量。

元组的长度是固定的，一旦创建就不能改变。元组是不可变的，创建后不能修改单个元素的值。你可以为元组的每个位置命名，如 `(name: String, age: Int64)`，但这只是类型标注，不能通过 `.name` 访问。

<CompareGroup>

<CompareWith lang="c">
C 不支持多返回值，通常通过返回结构体或使用输出参数（指针）来实现。这比元组解包更复杂繁琐。
</CompareWith>

<CompareWith lang="java">
Java 也不支持多返回值，需要定义一个包装类或使用数组来返回多个值。仓颉的元组在这方面更简洁。
</CompareWith>

<CompareWith lang="go">
Go 原生支持多返回值，语法为 `func minMax() (int, int)`，调用时用 `min, max := minMax()` 接收。仓颉使用元组实现类似效果。
</CompareWith>

<CompareWith lang="rust">
Rust 也使用元组返回多个值，语法和仓颉几乎相同。两者都支持元组解构 `let (a, b) = tuple` 和索引访问 `tuple.0`。
</CompareWith>

</CompareGroup>

<Note>元组至少要有两个元素。如果只需要一个值，直接返回该类型即可。</Note>
