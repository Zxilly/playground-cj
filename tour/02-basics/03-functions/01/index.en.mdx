# Defining Functions

Cangjie uses the `func` keyword to define functions, followed by the function name, parameter list, optional return type, and function body. The parameter list is enclosed in parentheses, with multiple parameters separated by commas; the return type is separated from the parameter list by a colon; the function body is enclosed in curly braces.

Parameters are defined as `name: Type`, for example `a: Int64` represents a parameter named `a` of type `Int64`. Function parameters are immutable and cannot be reassigned within the function body. If a function doesn't need to return a value, you can omit the return type or explicitly annotate it as `Unit`.

The code on the right defines three functions: `add` takes two integer parameters and returns their sum, `greet` prints a greeting without a return value, and `square` calculates the square of a number. Note that when calling a function, you only need to pass values without writing parameter names.

<CompareWith lang="c">
C uses the `int add(int a, int b)` style for function definition, with the return type written before the function name and parameter types after parameter names. C lacks modern named parameters or default parameter features.
</CompareWith>

<CompareWith lang="java">
Java uses the `public static int add(int a, int b)` style for method definition, requiring access modifiers and the `static` keyword. Java also doesn't support named parameters or default parameters.
</CompareWith>

<CompareWith lang="go">
Go uses the `func` keyword to define functions, with parameter types written after parameter names, like `func add(a int, b int) int`. Cangjie uses a colon to separate parameter names and types, which is the opposite of Go's style.
</CompareWith>

<CompareWith lang="rust">
Rust uses the `fn` keyword to define functions, and the syntax for parameters and return types is similar to Cangjie, both using colons as separators. However, Rust function names use snake_case, while Cangjie recommends camelCase.
</CompareWith>

</CompareGroup>

CJ:
// Define a function that takes two parameters and returns their sum
func add(a: Int64, b: Int64): Int64 {
    return a + b
}

// Define a function with no return value (return type is Unit)
func greet(name: String) {
    println("Hello, ${name}!")
}

// Define a function to calculate the square
func square(n: Int64): Int64 {
    return n * n
}

main() {
    let sum = add(10, 20)
    println("10 + 20 = ${sum}")
    
    greet("Cangjie")
    
    let result = square(7)
    println("The square of 7 is ${result}")
}
