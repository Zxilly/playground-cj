# var（可变绑定）

`var` 关键字用于创建可变绑定，绑定的值可以在后续代码中被重新赋值。当你需要计数、累积结果或更新状态时，使用 `var` 是合适的选择。

`var` 修饰的变量可以被多次赋值，每次赋值都会改变变量所持有的值。赋值操作符 `=` 用于修改变量的值，要求右侧表达式的类型是左侧变量类型的子类型。仓颉不支持链式赋值（如 `a = b = 0`），这可以避免常见的编码错误。

右侧代码展示了 `var` 的典型用法：`count` 从 0 开始，被多次修改；`message` 从 `"hello"` 改为 `"world"`。每次赋值后，我们打印变量的新值来观察变化。

<Note>一个好的编程实践是：默认使用 `let`，只有当你确定需要修改变量时才改用 `var`。这样可以减少意外修改带来的 bug。</Note>

<CompareWith lang="c">
C 的所有变量默认可变，无法在编译期强制实现不可变性。仓颉用 `var` 明确标记可变性，更清晰。

<CompareWith lang="java">
Java 默认所有变量可变，需要用 `final` 修饰才能实现不可变；仓颉反其道而行之，默认 `let` 不可变，需要 `var` 时才改。
</CompareWith>

<CompareWith lang="go">
Go 的变量默认可变，没有提供不可变变量的语法支持。仓颉通过 `let` 和 `var` 明确区分，让意图更清晰。
</CompareWith>

<CompareWith lang="rust">
Rust 使用 `let mut` 声明可变变量，仓颉使用 `var` 关键字，语义相同但语法更简洁直观。
</CompareWith>
