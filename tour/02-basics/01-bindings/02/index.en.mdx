# var (Mutable Bindings)

The `var` keyword is used to create mutable bindings, where the bound value can be reassigned in subsequent code. When you need to count, accumulate results, or update state, using `var` is the appropriate choice.

Variables modified with `var` can be assigned multiple times, and each assignment changes the value held by the variable. The assignment operator `=` is used to modify the variable's value, requiring the type of the right-hand expression to be a subtype of the left-hand variable's type. Cangjie does not support chained assignment (like `a = b = 0`), which helps avoid common coding errors.

<Note>A good programming practice is: use `let` by default, and only switch to `var` when you're certain you need to modify the variable. This reduces bugs from accidental modifications.</Note>

<CompareWith lang="java">

<CompareWith lang="go">
Go variables are mutable by default, with no syntax support for immutable variables. Cangjie clearly distinguishes between them through `let` and `var`, making intentions more explicit.
</CompareWith>

<CompareWith lang="rust">
Rust uses `let mut` to declare mutable variables, while Cangjie uses the `var` keyword, with the same semantics but simpler and more intuitive syntax.
