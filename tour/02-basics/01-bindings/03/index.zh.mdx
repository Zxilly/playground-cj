# 类型标注

在仓颉中，你可以在变量名后使用 `: Type` 显式指定变量的类型。类型标注让代码意图更清晰，编译器也会检查赋值的值是否与声明的类型一致，在编译期捕获类型错误。

仓颉是静态类型语言，每个变量都有确定的类型。常见的基本类型包括：`Int64`（64 位整数）、`Float64`（64 位浮点数）、`String`（字符串）、`Bool`（布尔值，只有 `true` 和 `false`）。类型名首字母大写是仓颉的命名约定。

类型标注的位置在变量名之后，这与 C/Java 的传统写法（类型在前）不同。这种设计的好处是：即使写了类型标注，变量名依然在前面，更突出"名字"这一核心信息，阅读代码时也能快速定位变量名。

<Note>类型标注是可选的。当变量有初始值时，编译器可以自动推断类型。但显式标注在某些场景下很有用：增加代码可读性、限制类型范围、或帮助编译器处理复杂情况。</Note>

</CompareWith>

<CompareWith lang="rust">
