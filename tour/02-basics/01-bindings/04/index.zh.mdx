# 类型推断

当变量有初始值时，编译器可以根据初始值的类型自动推断变量的类型，无需显式标注。这让代码更简洁，同时保持类型安全。类型推断是仓颉的类型系统特性之一。

右侧代码中，所有变量都没有类型标注，但编译器知道它们的类型：整数字面量推断为 `Int64`，浮点字面量推断为 `Float64`，字符串字面量推断为 `String`，布尔字面量推断为 `Bool`。这些是仓颉的默认类型推断规则。通过表达式计算得到的值也会被推断类型。

类型推断不是动态类型——变量的类型在编译时已经确定，运行时不会改变。如果你尝试把字符串赋值给推断为整数的变量，或者把浮点数赋值给推断为整数的变量，编译器都会报错。仓颉是静态类型语言，类型检查发生在编译期。

<Note>虽然类型推断很方便，但在某些情况下显式标注类型仍然有价值：提高代码可读性、避免意外推断为不期望的类型、或在类型推断不够精确时帮助编译器。</Note>

<CompareGroup>

<CompareWith lang="c">
C 不支持类型推断，变量声明时必须显式指定类型。仓颉的类型推断让代码更简洁，同时保持类型安全。
</CompareWith>

<CompareWith lang="java">
Java 仅在特定上下文（如 `var` 关键字在 Java 10+）支持类型推断；仓颉原生支持推断，适用于任何有初始值的变量。
</CompareWith>

<CompareWith lang="go">
Go 也支持类型推断（通过 `:=` 语法），整数字面量推断为 `int`（通常是 64 位），浮点数推断为 `float64`，与仓颉的默认推断规则相似。
</CompareWith>

<CompareWith lang="rust">
Rust 同样支持类型推断，但整数字面量默认推断为 `i32`（32 位整数），而仓颉默认推断为 `Int64`（64 位整数），在现代 64 位系统上更合适。
</CompareWith>

</CompareGroup>

试着添加一个新的没有类型标注的绑定，例如 `let z = 100 + 200` 或 `let sum = x + y`，猜猜它会被推断成什么类型。
