# let (Immutable Bindings)

The `let` keyword is used to create immutable bindings, associating a name with a value. Once bound, that name cannot point to a new value. This immutability is the default choice in Cangjie, helping you write safer and more predictable code.

In Cangjie, the basic form of variable definition is: `modifier variable_name: variable_type = initial_value`. The `let` modifier indicates that the variable is immutable, and the modifier is mandatory (cannot be omitted). Variables modified with `let` can only be assigned once, during initialization.

<Note>Local variables can be declared first and initialized later, but must be initialized before being referenced. Global variables and static member variables must be initialized at declaration.</Note>

<CompareGroup>

<CompareWith lang="c">
C has no native immutable variable concept; typically uses `const` for compile-time constants but lacks runtime guarantees. Cangjie's `let` provides guarantees at both compile and runtime.
</CompareWith>

<CompareWith lang="java">
Java uses the `final` modifier to declare immutable variables, with semantics similar to Cangjie's `let`, but requires explicit initialization.
</CompareWith>

<CompareWith lang="go">
Go uses `:=` to declare variables, and declared variables are always mutable with no concept of immutable variables. Cangjie's `let` provides immutability guarantees.
</CompareWith>

<CompareWith lang="rust">
Rust's `let` is also immutable by default with the same semantics as Cangjie, but Rust allows variable shadowing within the same scope, while Cangjie does not.
</CompareWith>
