# let（不可变绑定）

`let` 关键字用于创建不可变绑定，将一个名字与值关联起来。一旦绑定完成，该名字就不能再指向新的值。这种不可变性是仓颉的默认选择，有助于编写更安全、更易推理的代码。

在仓颉中，变量定义的基本形式是：`修饰符 变量名: 变量类型 = 初始值`。其中 `let` 修饰符表示该变量不可变，且必须修饰符是必要的（不能省略）。`let` 修饰的变量只能被赋值一次，即在初始化时赋值。

右侧代码定义了三个不可变绑定：`name` 绑定到字符串，`year` 绑定到整数，`pi` 绑定到浮点数。每个变量的类型都由初始值自动推断。如果尝试在第 10 行取消注释，编译器会报错——`let` 绑定的值不能被重新赋值。

<Note>局部变量可以先声明后初始化，但必须在被引用前完成初始化。全局变量和静态成员变量必须在声明时初始化。</Note>

<CompareGroup>

<CompareWith lang="c">
C 没有原生的不可变变量概念，通常用 `const` 修饰符表示编译时常量，但缺乏运行时保证。仓颉的 `let` 在编译和运行时都提供保证。
</CompareWith>

<CompareWith lang="java">
Java 用 `final` 修饰符声明不可变变量，语义与仓颉的 `let` 相似，但 Java 还需要显式初始化。
</CompareWith>

<CompareWith lang="go">
Go 使用 `:=` 声明变量，声明的变量始终可变，没有不可变变量的概念。仓颉的 `let` 提供了不可变性保证。
</CompareWith>

<CompareWith lang="rust">
Rust 的 `let` 也默认不可变，语义与仓颉相同，但 Rust 允许同作用域内变量遮蔽（shadowing），仓颉不允许。
</CompareWith>

</CompareGroup>
