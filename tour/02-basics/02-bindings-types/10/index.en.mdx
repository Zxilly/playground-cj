# Unit and Nothing

`Unit` represents the type of "no meaningful value," similar to `void` in other languages. `Unit` has only one value, written as `()`. Operations that only care about side effects, such as `println`, assignment expressions, and loop expressions, have `Unit` as their type.

`Nothing` is a more special type: it represents "there will be no value," and no expression of any type returns a value of `Nothing` type. Expressions that interrupt program flow—`break`, `continue`, `return`, and `throw`—all have `Nothing` type. After executing these expressions, subsequent code will not be executed.

<CompareWith lang="go">
Go has no direct `Unit` equivalent—functions without return values simply don't write a return type. Go also has no `Nothing` type and cannot express the semantics of "will not return."
</CompareWith>

<CompareWith lang="rust">
Rust's `()` type corresponds to Cangjie's `Unit`, and the `!` type (never type) corresponds to Cangjie's `Nothing`, with identical semantics.
</CompareWith>
