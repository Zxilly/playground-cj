# let (Immutable Binding)

`let` declares an immutable binding. Once initialized, it cannot be reassigned. This is the default choice for variable declarations in Cangjie—preferring immutable bindings makes code easier to understand and reason about.

Immutable binding only means that the binding itself cannot point to another value. If it holds a reference to a mutable object, the object's content can still be modified.

<CompareGroup>

<CompareWith lang="c">
C has no native immutable variable support; `const` is compile-time. Cangjie's `let` guarantees immutability at compile and runtime.
</CompareWith>

<CompareWith lang="java">
Java uses `final` for immutable variables, semantics similar to Cangjie's `let`.
</CompareWith>

<CompareWith lang="go">
Go uses `:=` short declaration or `var` to declare variables, all variables are mutable by default. Cangjie's `let` is similar to Go's constants, but constants must use `const`.
</CompareWith>

<CompareWith lang="rust">
Rust's `let` is also immutable by default, but allows shadowing within the same scope. Cangjie does not allow variable shadowing with the same name.
</CompareWith>

</CompareGroup>

<Note>Global variables and static member variables must be initialized at declaration time and cannot be assigned later.</Note>
