# let（不可变绑定）

`let` 声明一个不可变绑定。一旦初始化完成，就不能再对它重新赋值。这是仓颉变量声明的默认选择——优先使用不可变绑定可以让代码更易理解和推理。

不可变绑定只是说这个绑定本身不能指向另一个值。如果它持有的是可变对象的引用，对象的内容仍然可以被修改。

<CompareGroup>

<CompareWith lang="c">
C 没有不可变变量的原生支持，`const` 是编译时约束；仓颉的 `let` 在编译和运行时都保证不可变。
</CompareWith>

<CompareWith lang="java">
Java 用 `final` 修饰不可变变量，语义接近仓颉的 `let`。
</CompareWith>

<CompareWith lang="go">
Go 使用 `:=` 短声明或 `var` 声明变量，所有变量默认可变。仓颉的 `let` 类似 Go 的常量，但常量必须用 `const`。
</CompareWith>

<CompareWith lang="rust">
Rust 的 `let` 也默认不可变，但允许在同一作用域内遮蔽（shadowing），仓颉不允许同名变量遮蔽。
</CompareWith>

</CompareGroup>

<Note>全局变量和静态成员变量在声明时必须初始化，不能延迟赋值。</Note>
