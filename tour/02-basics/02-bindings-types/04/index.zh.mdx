# 类型推断

当变量有初始值时，编译器可以自动推断出它的类型，无需显式标注。这让代码更简洁，同时保持类型安全。

仓颉的推断规则很直观：整数字面量默认推断为 `Int64`，浮点数字面量推断为 `Float64`，双引号字符串推断为 `String`，`true` 和 `false` 推断为 `Bool`。推断发生在变量定义处，基于初始值的类型。

右侧代码中的四个变量都没有显式类型标注，编译器能准确推断出它们的类型。后续用这些变量进行的操作也依赖推断结果——`sum` 的类型由 `x + 10` 推断。试着把 `x` 的初始值从 `42` 改成 `3.14`，观察 `sum = x + 10` 会发生什么类型错误。

<CompareGroup>

<CompareWith lang="c">
C 不支持类型推断，变量声明时必须显式指定类型。仓颉的推断让代码更简洁。
</CompareWith>

<CompareWith lang="java">
Java 从 10 开始通过 `var` 支持类型推断，仓颉原生完全支持。
</CompareWith>

<CompareWith lang="go">
Go 同样支持类型推断。整数推断为 `int`（平台相关），浮点数推断为 `float64`。仓颉固定推断为 `Int64` 和 `Float64`，跨平台更一致。
</CompareWith>

<CompareWith lang="rust">
Rust 的类型推断可以跨语句、跨函数边界，较为激进。仓颉的推断主要基于初始值，局部性更强。
</CompareWith>
