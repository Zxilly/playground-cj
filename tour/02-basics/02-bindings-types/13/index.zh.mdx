# 赋值运算符

赋值运算符（`=`）用于给 `var` 声明的可变变量赋予新值。仓颉的一个重要安全设计是：赋值表达式的类型是 `Unit`，值是 `()`，而不是被赋值的值本身。这可以有效避免将判等运算符 `==` 误写成赋值运算符 `=` 的常见错误，例如 `if (a = 5)` 会在编译时报错。

仓颉不支持链式赋值（如 `a = b = 0`）。赋值时，总是先计算 `=` 右边的表达式，再计算 `=` 左边的表达式，最后进行赋值。这保证了求值顺序的可预测性。

仓颉提供复合赋值运算符，包括 `+=`、`-=`、`*=`、`/=`、`%=`、`**=`、`&&=`、`||=` 等，可以简化代码。复合赋值表达式同样要求操作数类型相同，且表达式的类型也是 `Unit`。复合赋值运算符也是非结合的，不支持连续使用。

右侧代码展示了基本赋值和复合赋值的使用方式。你可以尝试将复合赋值改写为普通赋值形式，体验两种写法的对比。

<Note>赋值表达式返回 `Unit` 是仓颉的安全特性，避免了 C/Java 中常见的 `if (a = 5)` 错误，这类错误在仓颉中会直接编译失败。</Note>
