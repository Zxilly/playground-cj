# 穷尽性检查

仓颉编译器会检查 `match` 表达式是否覆盖了所有可能的情况。对于 `enum` 类型，必须列出所有构造器；对于开放类型如 `Int64`，需要使用通配符 `_` 或绑定模式来处理剩余情况。

穷尽性检查有助于避免遗漏边界情况。如果 `match` 不完整，编译器会报错，这能在编译阶段发现潜在的 bug。这是模式匹配相比传统 `switch` 的一个重要优势。

对于 `enum`，列出所有构造器后就不需要 `_` 分支了，因为编译器知道已经覆盖了所有情况。但对于整数、字符串等类型，值域是开放的，必须用 `_` 收尾。
