# HashMap

`HashMap<K, V>` 将键映射到值，提供快速的键值查找。键类型 `K` 必须实现 `Hashable` 和 `Equatable<K>` 接口，值类型 `V` 可以是任意类型。

使用 `add(key, value)` 或下标赋值 `map[key] = value` 添加键值对，下标访问 `map[key]` 读取值，`contains(key)` 检查键是否存在，`remove(key)` 删除键值对。遍历时产出 `(key, value)` 元组，可以在循环中直接解构。

右侧代码演示了分数表的创建、查询、更新、删除操作，以及遍历所有键值对。

<CompareGroup>

<CompareWith lang="c">
C 没有内置的 HashMap，需要自己实现哈希表或使用第三方库。仓颉的 `HashMap` 内置高效，开箱即用。
</CompareWith>

<CompareWith lang="java">
Java 的 `HashMap<K, V>` 与仓颉类似，都使用哈希表存储键值对。两者都需要键实现哈希和相等比较。
</CompareWith>

<CompareWith lang="go">
Go 使用 `map[K]V` 作为哈希表，通过 `make(map[K]V)` 创建。仓颉的 `HashMap` 使用构造函数创建，`add` 方法添加键值对，也可以用下标赋值。
</CompareWith>

<CompareWith lang="rust">
Rust 的 `HashMap<K, V>` 与仓颉类似，但需要 `use std::collections::HashMap` 导入。仓颉的 `HashMap` 也显式导入，两者的遍历都产出元组。
</CompareWith>

</CompareGroup>
