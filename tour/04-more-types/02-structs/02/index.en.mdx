# Constructors and Value Semantics

A `struct` is a value type. Assigning a struct to another variable creates an independent copy; modifying the copy does not affect the original value.

A `struct` can define multiple `init` constructors (with different parameters) or define a primary constructor with the same name as the type. Parameters of the primary constructor can be decorated with `let` or `var`, simultaneously defining fields and constructor parameters.

<Note>
When defining a constructor, all instance member variables without initial values must be initialized, otherwise a compilation error will occur.
</Note>

<CompareGroup>

<CompareWith lang="c">
C structs have no constructor concept and require manual field initialization. Assignment copies the entire struct by value.
</CompareWith>

<CompareWith lang="java">
Java uses constructors to initialize objects, and multiple constructors can be defined (overloading). Assignment copies the reference, not the value.
</CompareWith>

<CompareWith lang="go">
Go's structs don't have constructor syntax; typically `NewXxx()` functions are defined to create instances. Assignment also copies.
</CompareWith>

<CompareWith lang="rust">
Rust's structs can have multiple `new` methods, and assignment also copies (unless moved).
</CompareWith>

</CompareGroup>
