# Struct Methods

Functions defined inside a `struct` are called member methods, which can access fields of the current instance through `this`.

Methods are called using dot notation on an instance, such as `c1.area()`. Methods can access all fields of the instance and can also call other methods. Static methods are decorated with `static` and called through the type name.

The `Circle` in the code on the right defines four methods: `area`, `circumference`, `describe`, and `isLargerThan`, demonstrating different method usages. `isLargerThan` accepts another `Circle` parameter for comparison.

<Note>
Regular methods cannot modify the value of `var` fields. If you need to modify fields in a method, use `mut func` (explained on the next page).
</Note>

<CompareGroup>

<CompareWith lang="c">
C has no concept of methods and typically defines functions with a struct pointer as the first parameter. Cannot access `this`.
</CompareWith>

<CompareWith lang="java">
Java defines methods inside classes and accesses instances through `this`. Visibility can be controlled with `private/public`.
</CompareWith>

<CompareWith lang="go">
Go's struct methods are defined outside the type, distinguished by "receiver" parameters, without a `this` keyword.
</CompareWith>

<CompareWith lang="rust">
Rust defines methods in impl blocks; `&self` indicates borrowing, `&mut self` indicates mutable borrowing, similar to Cangjie's regular methods and mut methods.
</CompareWith>
