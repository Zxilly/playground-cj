# Pattern Matching with Option

Using `match` with `Option` allows explicit distinction between `Some(v)` and `None` cases, with different handling logic in each branch. This is the most common and clearest way to handle `Option`.

`case Some(v)` binds the inner value of `Option` to variable `v`, which can be used in the branch code. `case None` handles the case when the value is empty.

The `where` guard can further refine matching conditions. For example, `case Some(s) where s >= 90` only matches `Some` where the inner value is greater than or equal to 90. Guards must be boolean expressions, placed after the pattern and before `=>`.
