# Exceptions vs Option

Use `Option` when absence is a normal outcome -- a lookup that finds no match returns `None`. Use exceptions when the situation is a genuine error the caller must handle.

```cangjie
func findScore(name: String): ?Int64 { ... }    // absence is normal
func withdraw(balance: Int64, amount: Int64): Int64 { ... }  // error is exceptional
```

`findScore` returns `?Int64` because a missing name is expected. `withdraw` throws because insufficient balance is an error condition.

<CompareWith lang="go">

</CompareWith>

<CompareWith lang="rust">
Rust uses `Option` for absence and `Result` for errors, matching Cangjie's split between `Option` and exceptions.
</CompareWith>
