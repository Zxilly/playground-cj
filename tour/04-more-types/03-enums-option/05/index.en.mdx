# Option and Null Safety

Cangjie provides multiple ways to handle `Option` values, making null-safe operations more convenient.

The `??` operator (coalescing operator) provides a default value when the value is `None`. The expression `opt ?? default` returns `v` when `opt` is `Some(v)`, and returns `default` when `opt` is `None`. This allows safe unwrapping of Optional values.

The `getOrThrow()` method directly unwraps the value, returning `v` when the value is `Some(v)`, and throwing an exception when it's `None`. This is suitable when you're certain the value must exist.

When the context clearly requires an `Option<T>` type, you can directly pass a `T` type value, and the compiler will automatically wrap it with `Some`. For example, a function returning an `Int64` type value can directly return the number, which will be automatically wrapped as `Some`.

<CompareGroup>

<CompareWith lang="c">
C doesn't have a coalescing operator; it requires manual ternary operators or if-else for null handling. Cangjie's `??` is more concise.
</CompareWith>

<CompareWith lang="java">
Java objects can be `null` and require explicit checking. Cangjie's `Option` and `??` operator provide safer null value handling.
</CompareWith>

<CompareWith lang="go">
Go's `if err != nil` pattern is more verbose than Cangjie's `??`, but more explicit. Go doesn't have a similar auto-wrapping mechanism.
</CompareWith>

<CompareWith lang="rust">
Rust's `unwrap_or()` method has the same functionality as Cangjie's `??` operator, but `??` syntax is more concise.
</CompareWith>

</CompareGroup>

<Note>The `??` operator requires the default value on the right side to have the same type as the inner type of `Option`.</Note>
