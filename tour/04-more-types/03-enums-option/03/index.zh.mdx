# 递归枚举

枚举构造器的参数可以是该枚举类型本身，形成递归数据结构。这种枚举常用于表示树形结构、表达式等。

右侧的 `Expr` 枚举表示一个算术表达式：`Num` 是叶子节点（基本数值），`Add` 和 `Mul` 是分支节点，包含两个子表达式。

处理递归枚举通常需要递归函数。`eval` 函数递归计算表达式的值：遇到 `Num` 直接返回数值，遇到 `Add` 或 `Mul` 则递归计算子表达式后相加或相乘。

`show` 函数将表达式转换为可读的字符串形式，同样采用递归处理。

<CompareGroup>

<CompareWith lang="c">
C 不支持递归数据结构的直接表示，需要手动管理指针和内存。仓颉的递归枚举简洁安全。
</CompareWith>

<CompareWith lang="java">
Java 没有枚举的递归版本；通常使用类层级和多态替代。仓颉的递归枚举更简洁直观。
</CompareWith>

<CompareWith lang="go">
Go 无法直接定义递归枚举，通常需要使用接口和结构体的组合，代码更冗长。
</CompareWith>

<CompareWith lang="rust">
Rust 的递归枚举需要在构造器参数前加 `Box` 包装，因为枚举大小必须固定。仓颉会自动处理，语法更简洁。
</CompareWith>
